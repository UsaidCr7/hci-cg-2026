Clean Comparison (Real-Life Coffee Touchscreen Counter )

Model-based evaluation: what each model explains vs. predicts
Model Category What it Explains What it Predicts
Norman Cognitive / descriptive Understanding of buttons, errors, system feedback, gulfs of execution and evaluation ❌ Nothing (cannot predict how fast the order is made)
GOMS Predictive / analytic Task structure: selecting coffee type, size, extras, payment ✔ Relative efficiency of menu paths (e.g., shortcut buttons vs. scrolling)
KLM Predictive / quantitative Low-level actions: tap, swipe, hand movement, system wait ✔ Execution time to complete the order

KLM Operators (Execution-Level Primitives for Coffee Ordering)

In a touchscreen kiosk, KLM can model the execution of taps, swipes, hand movements, and waiting. The user is assumed to be familiar with the interface.

KLM operators for the coffee kiosk
Operator Name Meaning Example
K Keystroking Pressing keys on a virtual keypad Entering loyalty number on screen
B Button press Touching screen buttons Select "Latte" or "Size: Medium"
P Pointing Moving finger to touch target Move finger to "Add Extra Shot" checkbox
H Homing Switching between keyboard (if used) and touchscreen Keyboard → touch screen for payment
D Drawing Drag gestures on touchscreen Swipe to scroll menu
M Mental preparation Thinking before choosing an option Deciding which size and extras to select
R System response Waiting for system feedback Screen loading new menu options

Worked Example: Ordering a Latte

Task: User orders a medium latte with an extra shot, pays, and confirms.

  1. Think which coffee and size to select
  2. Move hand to touchscreen
  3. Tap "Latte"
  4. Tap "Medium"
  5. Tap "Extra Shot"
  6. Move to payment keypad if needed
  7. Enter card number / tap "Pay"
  8. Wait for confirmation screen

One possible KLM operator sequence:

M P B[Latte] 
P B[Medium] 
P B[Extra Shot] 
H[keypad] M K[CardNumber] 
P B[Pay] 
R

This models only execution-level actions. Planning and decision-making are not included.

Key Takeaways

KLM operators (K, B, P, H, D, M, R) can be applied to real-world tasks like touchscreen ordering to predict execution time for expert users.

GOMS provides task-level structure, Norman explains cognitive understanding, errors, and feedback—together they help analyze usability.
